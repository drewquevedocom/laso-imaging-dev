

## Plan: Google OAuth + Account Type Selection on Signup

### What we'll build

1. **Google OAuth "Sign in with Google" button** on both the `/signup` page and `/auth/customer` page using Lovable Cloud's managed Google OAuth (no API keys needed)

2. **Account type selector** — a required radio group asking "I am a..." with three options: Reseller, Engineer, Buyer. This maps to the existing `customer_type` column on `customer_profiles` (currently defaults to `'general'`).

3. **Phone field made required** on the signup form (it's already there but optional). The phone number will be stored on `customer_profiles` for future contact.

### Important note on SMS verification

Lovable Cloud does not support SMS-based OTP verification for account signup. The standard verification method is email confirmation. Your Telnyx integration is set up for outbound admin SMS (sending messages to leads), not for auth verification codes. To add true SMS OTP verification would require a Twilio integration with custom edge function logic, which is a significant addition.

**Recommended alternative:** Keep email verification for account creation, and use the collected phone number for follow-up SMS via your existing Telnyx integration (e.g., welcome SMS after signup).

### Flow

- **Email signup**: User fills form including account type → creates account → email verification → portal access
- **Google OAuth**: User clicks "Continue with Google" → Google auth → redirected back → profile auto-created with `customer_type = 'general'` → on first portal visit, prompt to select account type (or we set it post-OAuth via a profile completion step)

For Google OAuth users, since we can't collect account type before the OAuth redirect, we'll update the profile after redirect using a brief "complete your profile" prompt on the portal dashboard.
